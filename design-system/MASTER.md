## Component Rules

### Button
- Use `Pressable` (not `TouchableOpacity`) for better press-state control
- Animate: `scale(0.97)` on press, `scale(1.0)` on release — 150ms ease-out
- Minimum height: `44pt` (touch target requirement)
- Always include `accessibilityRole="button"` and `accessibilityLabel`
- Loading state: disable + show `ActivityIndicator`, never hide text abruptly

### Card
- `borderRadius: 16` — keep
- Shadow color: `#1A1035` (not black) for warmer, more premium shadow
- Add `accessibilityRole="none"` on decorative cards
- Interactive cards: wrap in `Pressable` with `accessibilityRole="button"`

### Badge
- Minimum tap area if tappable: 44×44pt (use `hitSlop`)
- Font size minimum: 12px
- Never use color alone to convey meaning — include icon or text label

### Input
- Label always visible (not placeholder-only)
- Error message below field, not at top
- Minimum height: 44pt
- Use `textContentType` and `autoComplete` for autofill support
- Show/hide toggle on password fields

### Icons (Ionicons)
- Decorative icons: `accessibilityElementsHidden={true}` (iOS) / `importantForAccessibility="no"` (Android)
- Meaningful icons: `accessibilityLabel="description"`
- Never use emoji as structural/navigation icons

## Anti-Patterns to Avoid

- ❌ Raw hex values hardcoded in screen files — always import from `Colors`
- ❌ `fontSize` below 12 anywhere
- ❌ `TouchableOpacity` for new interactive elements — use `Pressable`
- ❌ Icons without `accessibilityElementsHidden` or `accessibilityLabel`
- ❌ Animations on `width`/`height` — only `transform` and `opacity`
- ❌ Shadow color `#000000` — use `#1A1035` for warmer shadows
- ❌ Mixing icon styles (use Ionicons consistently, one stroke weight)
